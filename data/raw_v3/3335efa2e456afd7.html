Patch releases

Based on the GitLab Maintenance Policy, patch releases include bug fixes for the current stable released version of GitLab and security fixes for the two previous monthly releases in addition to the current stable release.

The cadence of patch releases is twice a month on the Wednesdays before and after the monthly release week. These are best-effort dates and they might be subject to change.

GitLab Maintained Versions

The GitLab Maintenance Policy maintains the current stable release and the previous two monthly releases. Refer to currently maintained versions.

More information

There are a number of ways you can find out about the latest releases and their features:

  • Go to the GitLab project releases page to learn about recent features in our official GitLab releases.
  • Visit the releases blog to view release posts, including patch release posts.
  • Use the What's new feature while you're using GitLab to discover new features.
  • Go to the GitLab.com release page for information about the features and current release on GitLab.com.
  • Check the changelogs:
    • GitLab Community Edition
    • GitLab Enterprise Edition
    • GitLab Runner
  • Compare the feature changes between two different releases by using the Release Feature & Deprecation Overview page. Due to the large number of improvements in each release, you can export the information to CSV.
  • GitLab Quarterly Launch.
  • Release Blog Archives - Access all previous GitLab release posts, organized by version number and date.

To learn how to update your self-managed instance so you can take advantage of new features, see the update documentation.

Changes prior to GitLab 8.0 are available in the archive.

Find out what we're planning to release in future by:

  • Viewing the most important features of upcoming releases on the upcoming releases page.
  • Viewing upcoming features by product tier.

To learn more about how our releases work, see the monthly release template in the release tools repository.
